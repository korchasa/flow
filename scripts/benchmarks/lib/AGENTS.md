# Cursor Agent: Internal Mechanics & Integration

## Overview

`cursor-agent` — это CLI-интерфейс для взаимодействия с AI-агентом AssistFlow. Он поддерживает автономное выполнение задач, использование инструментов (чтение/запись файлов, запуск команд) и интерактивный режим.

## Output Formats

При использовании флага `--print` доступны следующие форматы вывода (`--output-format`):

1. **text** (по умолчанию): Человекочитаемый вывод. Логи инструментов и ответы ассистента выводятся в поток по мере появления.
2. **json**: Весь вывод (логи инструментов и финальный результат) представляется в виде **одного JSON-объекта**, который печатается в stdout непосредственно перед завершением процесса.
3. **stream-json**: Потоковый вывод JSON-объектов (событий) по мере выполнения.

## JSON Format Structure

В режиме `--output-format json` агент выдает объект следующей структуры:

```json
{
  "type": "result",
  "subtype": "success",
  "is_error": false,
  "duration_ms": 4032,
  "result": "Привет! Чем я могу вам помочь сегодня?",
  "session_id": "d5a66525-6040-4010-b0e4-b35dba72ccf2",
  "messages": [
    { "type": "assistant", "content": "..." },
    { "type": "user", "content": "..." }
  ]
}
```

### Key Fields:

- `session_id`: Уникальный идентификатор чата. Используется для продолжения диалога через `--resume <session_id>`.
- `result`: Финальный текстовый ответ ассистента.
- `messages`: Полная история диалога в текущей сессии.
- `subtype`: Статус завершения (`success`, `error`, `input_required`).

## Lifecycle & Resume

1. **Первый запуск**: Агент выполняет шаги, пока не решит задачу или не потребует ввода пользователя. Перед выходом печатает JSON.
2. **Input Required**: Если агенту нужен ввод, он завершается с соответствующим статусом в JSON.
3. **Resume**: Для продолжения вызывается `cursor-agent <new_prompt> --resume <session_id>`. Агент загружает историю из облака/базы по `session_id` и продолжает работу.

## Integration Notes for Benchmarks

- **Parsing**: В режиме `json` не нужно парсить вывод "на лету". Достаточно дождаться завершения процесса и распарсить последнюю строку (или весь буфер) как один JSON-объект.
- **State**: Состояние агента (история сообщений) хранится на стороне сервера/сессии, поэтому `SpawnedAgent` должен только извлекать `session_id` для последующих вызовов.
- **Mocks**: При тестировании через бенчмарки инструменты мокаются через Cursor Hooks, что позволяет имитировать побочные эффекты без реального выполнения опасных команд.
